Github学习笔记

发表于 2015-10-13

一、安装和设置

安装git

  1. 前往 homebrew网站 安装homebrew

    ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

  2. 安装git，在终端输入

    $ brew install git

设置

  1. 设置name和email

    $ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"

    –global表示所有的Git仓库都会使用这个配置

二、本地Repository

  1. 建立一个空目录作为本地repository

    $ mkdir learngit
$ cd learngit
$ pwd
/Users/michael/learngit

  2. 将该目录变成Git可以管理的repository

    $ git init

    这样，目录下多了一个.git目录，这是Git用来跟踪管理版本库的

    $ ls -ah

  3. 在本地repository中新增文件，并提交；修改文件，并提交

    • 新增文件

      $ vim test.txt

    • 添加文件至暂存区（等待提交）

      $ git add test.txt

    • 提交修改: -m后面的字符串是本次提交的说明

      $ git commit -m "add test.txt"

    • 修改文件

    • 查看状态

      $ git status
On branch master
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

modified:   test.txt

no changes added to commit (use "git add" and/or "git commit -a")

    • 查看文件修改内容

      $ git diff test.txt
diff --git a/test.txt b/test.txt
index 6de7b8c..2e0c855 100644
--- a/test.txt
+++ b/test.txt
@@ -1 +1,3 @@
-This is a file.
+That is a file.
+
+bob 2015.10.10

  4. 管理版本

    • 查看提交日志

      $ git log

    • 退回之前的版本

      $ git reset --hard HEAD^

      HEAD表示当前版本，HEAD^表示上一个版本，HEAD^^表示上上个版本，HEAD~100表示往上100个版本

      当然，也可以通过commit id来指定回退到哪个版本

      $ git reset --hard cb926e7ea50ad11b8f9e909c05226233bf755030

    • 查看命令历史

      $ git reflog

  5. 撤销修改

    • 丢弃工作区的修改，回到上一次 git add 或者 git commit 时的状态

      $ git checkout -- test.txt

    • 撤销已经被添加到暂存区中的修改

      $ git reset HEAD test.txt

  6. 删除文件

    • 将删除操作添加到暂存区

      $ git rm test.txt

三、远程Repository

  1. SSH Key

    如果~目录下没有.ssh文件夹，或其中没有id_rsa或id_rsa.pub文件

    $ ssh-keygen -t rsa -C "youremail@example.com"

    将公钥id_rsa.pub添加到Github的SSH Keys中

  2. 添加远程库

    • 在Github创建一个Repository

    • 将本地Repository与远程Repository

      $ git remote add origin git@github.com:在Github的用户名/Repository名称.git

  3. 将本地库推送至远程库

    $ git push -u origin master

    由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

    之后可以直接push本地库到远程库

    $ git push origin master

  4. 克隆远程库

    $ git clone git@github.com:在Github的用户名/Repository名称.git

$ git checkout -b dev origin/dev

  5. 查看远程库的信息

    $ git remote
$ git remote -v
